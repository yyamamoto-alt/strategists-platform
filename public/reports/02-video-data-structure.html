動画講座 複数動画URL対応レポート

2026年3月17日 Strategists LMS
Summary

動画講座が「1レッスン = 1動画URL」だった制約を解消し、1レッスンに対して複数の動画を構造化データとして管理できるようにしました。 従来の HTML 内 iframe ベタ打ち + 正規表現パースという脆い実装を廃止し、JSONB カラムによるシンプルかつ拡張性のある設計に移行しています。 既存データとの完全な後方互換性を維持しており、段階的な移行が可能です。

1. 変更の背景

従来の問題

Strategists LMS では、ケース面接対策・フェルミ推定対策などの動画講座を提供しています。 これらの講座は1レッスンあたり 5〜7 本の動画で構成されていますが、従来のデータ構造では以下の問題がありました。

  • lessons テーブルの video_url フィールドが 1つだけ であり、複数動画を直接管理できなかった
  • 複数動画は markdown_content（実態は HTML）内に YouTube の <iframe> をベタ打ちして埋め込んでいた
  • フロントエンドの MultiVideoPlayer コンポーネントが HTML を 正規表現でパースして動画セクションを抽出する設計になっていた
  • 管理者が HTML コンテンツを編集すると、正規表現パースが壊れるリスクが常に存在していた

つまり、「データ構造」と「表示ロジック」の両方に問題があり、管理者にとっても開発者にとっても不安定な状態でした。 特に、HTMLの記述ルールを管理者が把握していないと動画が正しく表示されないという運用リスクは、LMSの信頼性を損なう原因になり得ます。

2. 設計判断

複数動画の管理方法について、以下の3つの選択肢を検討しました。 それぞれのトレードオフを評価した結果、Option A（JSONBカラム追加）を採用しています。

Option A: JSONBカラム追加 採用

lessons テーブルに video_urls JSONB カラムを追加し、 [{title, url, duration_minutes, description}, ...] の配列構造で動画情報を格納する。

シンプルな実装。JOIN不要で1クエリで完結
並び順が配列インデックスで自然に表現できる
マイグレーションが1カラム追加のみで低リスク
RDB正規化の観点からは外れる
ただし、1レッスンあたり 5〜7 要素という規模では正規化のメリット（検索・集計）がほぼ不要であり、JSONBで十分。
Option B: 別テーブル（lesson_videos） 不採用

lesson_videos テーブルを新設し、lesson_id で外部キー参照する正規化設計。

RDB正規化の正攻法
5〜7本の動画に対してリレーション管理のオーバーヘッドが大きい
並べ替えに sort_order カラムの管理が別途必要
表示時にJOINが必要になりクエリが複雑化
Option C: 既存フィールド活用（現状維持） 不採用

markdown_content 内の HTML + 正規表現パースをそのまま使い続ける。

変更不要
HTMLパースの脆さを解消できない（根本問題が残る）
管理者のHTML編集ミスで動画表示が壊れるリスクが残る

3. 実装内容

3-1. DBマイグレーション

lessons テーブルに video_urls カラムを追加するシンプルなマイグレーションです。 デフォルト値として空配列 [] を設定しているため、既存レコードへの影響はありません。 

-- supabase/migrations/20260316_lesson_video_urls.sql

ALTER TABLE lessons
  ADD COLUMN IF NOT EXISTS video_urls JSONB DEFAULT '[]'::jsonb;
実行状況
マイグレーションは Supabase Management API 経由で実行済みです。本番環境に反映されています。

3-2. 型定義

packages/shared-db/src/types.ts に LessonVideo インターフェースを追加し、 既存の Lesson 型に video_urls フィールドを追加しました。 

// packages/shared-db/src/types.ts

export interface LessonVideo {
  title: string;              // "動画1: フェルミ推定の基礎"
  url: string;                // YouTube/Google Drive URL
  duration_minutes?: number;  // 個別動画の長さ（分）
  description?: string;       // 動画の説明文
}

export interface Lesson {
  // ...既存フィールド
  video_urls?: LessonVideo[];  // 追加
}

title と url が必須、duration_minutes と description はオプショナルとしています。 これにより、最小限の入力で動画を登録でき、詳細な情報は後から追加できる柔軟な設計になっています。

3-3. API更新

apps/lms/src/app/api/lessons/[id]/route.ts の PATCH ハンドラを更新しました。 

// apps/lms/src/app/api/lessons/[id]/route.ts

// allowedFields に video_urls を追加
const allowedFields = [
  'title', 'content', 'video_url',
  'markdown_content', 'sort_order',
  'video_urls',  // 新規追加
];

// バリデーション
if (body.video_urls !== undefined) {
  if (!Array.isArray(body.video_urls)) {
    return NextResponse.json(
      { error: 'video_urls must be an array' },
      { status: 400 }
    );
  }
  for (const v of body.video_urls) {
    if (!v.title || !v.url) {
      return NextResponse.json(
        { error: 'Each video must have title and url' },
        { status: 400 }
      );
    }
  }
}

3-4. 管理画面 UI

apps/lms/src/app/admin/contents/[id]/page.tsx に「複数動画管理」セクションを追加しました。 動画レッスンの編集時に以下の操作が可能です。

  • タイトル / URL / 所要時間（分）を行ごとに入力
  • 「動画を追加」ボタンで新しい行を追加
  • ↑↓ボタンで動画の並べ替え
  • ×ボタンで動画の削除

既存の単一 video_url 入力フィールドもフォールバック用として残しています。 これにより、単一動画のみのレッスンでは従来通りの簡単な入力が可能です。

3-5. 表示コンポーネント

components/content/multi-video-player.tsx に新しい Props videos?: LessonVideo[] を追加しました。 表示ロジックは以下の優先順位で動作します。

  1. videos 配列が渡された場合 → 構造化データをそのまま使用（パース不要）
  2. videos がない場合 → 従来の parseVideoSections(mainVideoUrl, htmlContent) にフォールバック
  3. 単一動画のみの場合 → シンプルな VideoPlayer にフォールバック

3-6. 受講者向け表示

lesson-player-client.tsx では、レッスンデータに video_urls が存在する場合はそちらを優先して MultiVideoPlayer の videos props に渡します。 

// lesson-player-client.tsx（概要）

{lesson.video_urls && lesson.video_urls.length > 0 ? (
  // 構造化データを優先
  <MultiVideoPlayer
    videos={lesson.video_urls}
  />
) : (
  // 従来パターンにフォールバック
  <MultiVideoPlayer
    mainVideoUrl={lesson.video_url}
    htmlContent={lesson.markdown_content}
  />
)}

これにより、完全な後方互換性を維持しています。 既存の動画レッスンは一切変更なしで従来通り動作し、video_urls に構造化データを登録したレッスンから順に新しい表示ロジックに切り替わります。

4. 使い方ガイド

管理者が動画講座を登録する手順

  1. 管理画面にログインし、サイドバーから 「教材管理」 を開く
  2. 対象のコンテンツ（例: ケース面接対策講座）を選択する
  3. 動画レッスンを開く（種類が「動画」のもの）
  4. 「複数動画管理」セクションで「動画を追加」ボタンをクリック
  5. タイトルを入力（例: 「第1講 フェルミ推定の基礎」）
  6. URLを入力（YouTube または Google Drive の共有URL）
  7. 所要時間（分）を任意で入力
  8. 動画の数だけ手順 4〜7 を繰り返す
  9. 並び順を ↑↓ ボタンで調整し、「保存」をクリック

既存データの移行について

移行は任意・段階的に実施可能

既存の動画レッスンはフォールバックロジックにより引き続き正常に動作します。 移行を急ぐ必要はありません。

移行する場合は、管理画面で既存の HTML コンテンツ内から動画 URL をコピーし、「複数動画管理」セクションに登録してください。 構造化データが登録されると、そちらが優先して使用されます。
