教材データ修復

DB 重複レッスン統合

同一コンテンツが3つのカリキュラム（スタンダード/ライト、ミニマム、選コミュ）に重複して存在していた問題を解消。

  • 対象: 7つのcontent_id、計13件の重複レッスン
  • 処理: 各コンテンツにつき1レッスンのみ保持、残りのcontent_idをNULLに
  • カリキュラム構造は維持（レッスン自体は削除せず）
DB 欠落コンテンツ補完（Notion連携）

レッスンが0件だったcontentsに対してNotionからデータを取得し、レッスンを作成。

動画講座データ構造変更

DB 複数動画URL対応

動画講座が複数動画（5〜7本）を持てるようにデータ構造を拡張。

変更内容

  • DB: lessonsテーブルに video_urls (JSONB) カラム追加
  • 構造: [{title, url, duration_minutes, description}]
  • 管理画面: 動画レッスン編集に「複数動画管理」UI追加（タイトル/URL/分数、追加・削除・並替え）
  • 表示: video_urlsがあればプレイリスト表示、なければ従来のHTMLパースにフォールバック
既存の動画データは引き続き動作します。新しい動画は管理画面の「複数動画管理」セクションから追加可能。

モバイル対応

モバイル サイドバー ハンバーガーメニュー化
  • デスクトップ (1024px以上): 従来通りサイドバー常時表示
  • モバイル/タブレット: サイドバー非表示、左上のハンバーガーボタンで開閉
  • 半透明オーバーレイ付きスライドインアニメーション
  • メニュー項目タップ後に自動クローズ
  • 外側タップでも閉じる
モバイル モバイルヘッダー追加
  • 高さ56pxの固定ヘッダー（モバイル時のみ表示）
  • 左: ハンバーガーボタン / 中央: 「Strategists LMS」
  • メインコンテンツはフル幅表示

セキュリティ強化

SEC HTTPヘッダー
  • X-Content-Type-Options
  • X-Frame-Options
  • X-XSS-Protection
  • Referrer-Policy
  • HSTS
SEC レート制限
  • ログイン: 5回/分
  • Enrollment Webhook: 20回/分
  • Slack Webhook: 30回/分
  • 超過時 → 429エラー
SEC エラーメッセージ安全化

Supabaseの内部エラーメッセージがAPIレスポンスに漏れていた問題を修正。一般的な日本語メッセージに置換し、詳細はサーバーログのみに出力。

UX基盤整備

UX Toast通知システム

操作結果のフィードバックが皆無だった問題を解消。

  • 4タイプ: success（緑）/ error（赤）/ warning（黄）/ info（青）
  • 自動消去: 通常3秒、エラー5秒
  • 画面右下にアニメーション表示
  • 適用済: レッスン保存・削除時
UX エラーバウンダリ (error.tsx)

Server Componentでエラー発生時に白画面になる問題を解消。3箇所に追加:

  • グローバル (app/error.tsx)
  • メインセクション (app/(main)/error.tsx)
  • 管理画面 (app/admin/error.tsx)

日本語メッセージ + 「再度読み込み」ボタン付き。

UX 確認ダイアログ

ブラウザ標準の window.confirm() をカスタムモーダルに置換。

  • オーバーレイ付きモーダル表示
  • 削除時は赤ボタン（destructiveモード）
  • Escapeキーでキャンセル
  • 適用済: レッスン削除時
